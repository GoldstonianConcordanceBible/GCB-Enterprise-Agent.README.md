# UX Playbooks — GCB

---

## 🎨 PLAYBOOK 1: Clear Path Design

Every user path must answer:

1. Where am I?
2. What is this?
3. What do I do next?

If those are unclear, the UX failed.

---

## 🎨 PLAYBOOK 2: Theological Learning Flow

Move the user through:

- Curiosity
- Revelation
- Structure
- Participation
- Commitment

This means:
- Shorts pull attention
- Long-form content deepens conviction
- GitHub/Substack establishes authority
- Email captures relationship

---

## 🎨 PLAYBOOK 3: Friction Reduction

Reduce:
- Too many clicks
- Too many competing links
- Unclear calls to action
- Dense unexplained repo structures

Simplify:
- One primary CTA per asset
- One next step per page

---

## 🎨 PLAYBOOK 4: Curriculum Experience

For SydTek Scholars and GCB learning systems:

- Video = lecture
- Description = syllabus layer
- Pinned comment = discussion prompt
- GitHub = archive + evidence
- Email = retention layer

---

## 🎨 PLAYBOOK 5: Experience Consistency

Every platform must reinforce:

- Goldstonian Concordance Bible (GCB)
- Mirror → Water → Fire Doctrine
- 81-Book Ethiopian Canon
- SydTek Scholars

No terminology drift allowed.
